TRAVELING SALESMAN PROBLEM

Game Manual

Introduction

Welcome to the TSP Game, an interactive visualization of the classic Traveling Salesman Problem. This educational game allows you to explore one of the most famous problems in computer science and operations research through an engaging 3D interface.

What is the Traveling Salesman Problem?

The Traveling Salesman Problem (TSP) asks: "Given a list of cities and the distances between each pair of cities, what is the shortest possible route that visits each city exactly once and returns to the origin city?"

This is a famous NP-hard problem in combinatorial optimization, studied in operations research and theoretical computer science.

Game Objectives

In this game, you will:

  • Create routes by selecting countries/cities on a 3D globe
  • Try to find the shortest possible route that visits all locations
  • Learn about optimization algorithms through interactive play
  • Compare your manual solutions with computer-generated ones

Controls & Interface

Globe Navigation

  • Rotate: Click and drag on the globe
  • Zoom: Use mouse wheel or pinch gesture
  • Auto-rotate: Toggle with the AUTO ROTATE button

Node Selection

  • Hover over a node to see country information
  • Click on a node to add it to your route
  • Nodes glow brighter when hovered or selected

Control Buttons

  • AUTO ROTATE: Toggle automatic rotation of the globe
  • NEW GAME: Reset the game and start over
  • OPTIMIZE: Generate an optimized route using algorithms
  • COMPLETE ROUTE: Finish your route by connecting back to the starting point
  • DESIGN THINKING: Learn about the game's design principles

Pro Tip

When selecting nodes, try to think ahead about the overall route shape. Sometimes a slightly longer individual connection can result in a much shorter overall route!

Game Mechanics

Route Building

As you select countries, the game will automatically connect them with arcs. These arcs represent the path your "salesman" will travel.

Distance Calculation

The game calculates distances between points using great-circle distance (the shortest distance between two points on a sphere). This represents the real-world distance between locations.

Energy Units

Energy units are calculated based on distance traveled. They represent the resources consumed during your journey.

Optimization Algorithm

The "Optimize" button uses a nearest neighbor algorithm, which is a greedy approach to solving TSP. While not guaranteed to find the optimal solution, it's computationally efficient and produces reasonably good routes.

Strategy Tips

  • Start by identifying clusters of nodes that are close together
  • Try to avoid crossing your own path
  • Consider the return trip to your starting point
  • Compare your manual solutions with the algorithm's solution
  • Experiment with different starting points

Educational Value

This game demonstrates several important concepts in computer science and mathematics:

  • NP-hard problems and computational complexity
  • Heuristic algorithms vs. exact solutions
  • Graph theory and optimization
  • Spatial reasoning and 3D visualization
  • Great-circle distance calculation
